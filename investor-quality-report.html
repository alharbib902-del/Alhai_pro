Security Architecture

Data at Rest

  • SQLCipher database encryption — all local data encrypted with per-device keys via sqlcipher_flutter_libs
  • FlutterSecureStorage — tokens, encryption keys stored in Android EncryptedSharedPreferences / iOS Keychain
  • Web storage obfuscation — per-session random key + XOR for web platform fallback
  • Automatic key generation — 32-byte cryptographically random keys via Random.secure()

Data in Transit

  • Certificate pinning — SHA-256 fingerprint validation on Supabase and API connections
  • Secure HTTP client — Dio-based with badCertificateCallback enforcement
  • Quarterly certificate rotation check — automated CI cron job verifies SSL expiry dates
  • Dart-define secrets — no hardcoded API keys; injected at build time from GitHub Secrets

Input Validation & Sanitization

  • InputSanitizer class — XSS, SQL injection, command injection, path traversal protection
  • SanitizedTextFormField widget — auto-sanitizes all user input on save/submit
  • Comprehensive validators — phone, email, IBAN, barcode, price, form fields
  • Dangerous content detection — real-time script/SQL pattern matching

Authentication & Access

  • Supabase Auth with WhatsApp OTP verification
  • PIN-based POS access with brute-force protection
  • Biometric authentication support (fingerprint/face)
  • Session management with expiry tracking
  • Security audit logging for compliance
  • Role-based permissions (manager approval workflows)

CI Security Scanning

  • Secret scanning: API keys, passwords, private keys, JWT tokens, AWS/Firebase/Slack credentials
  • Dependency vulnerability audit (weekly)
  • Strict static analysis on security-critical packages
  • .env file tracking detection

ZATCA E-Invoicing Compliance

Dedicated alhai_zatca package with 36 source files and 216 test cases implements full Saudi Arabia e-invoicing requirements.

Capabilities

  • UBL 2.1 XML generation — invoice builder, line builder, tax total builder
  • ECDSA digital signing — SHA-256 invoice hashing, XAdES signature structure
  • TLV QR code encoding — Base64-encoded seller, VAT, timestamp, totals
  • CSID onboarding — CSR generation, certificate parsing, renewal service
  • Compliance checker — validates invoices against ZATCA business rules
  • Invoice chaining — cryptographic hash chain for sequential integrity
  • XML canonicalization — C14N for deterministic signing
  • Offline queue — stores invoices locally when API unavailable
