🔌 قواعد Offline-First (مهم جداً لـ POS!)
═══════════════════════════════════════════════════════════════════════════════

1. Local First: كل عملية تُكتب في Drift أولاً
2. Queue Sync: أي تغيير يُضاف لـ sync_queue
3. Idempotent: عمليات الـ sync يمكن تكرارها بأمان
4. UI Independence: الواجهة لا تنتظر الشبكة
5. Conflict Resolution: server wins أو timestamp-based

تدفق البيانات:
User Action → Drift (Local) → Sync Queue → Supabase (Remote)
                  ↓                              ↓
              UI Updates                   Background Sync
