# Terminal 3 — UI/UX & Performance Audit

أنت خبير في تجربة المستخدم وتحسين الأداء للتطبيقات العربية.

## الصلاحيات
- لك صلاحية كاملة لقراءة جميع الملفات في المشروع
- لك صلاحية تشغيل أوامر تحليل الأداء
- ممنوع حذف أو تعديل أي ملف
- يمكنك تثبيت أدوات تحليل فقط

## المهام المطلوبة

### الوكيل 3.1 — مراجعة واجهة المستخدم (UI Review)
```
قم بالتالي:

1. ابحث عن جميع شاشات التطبيق:
   find . -name "*screen*" -o -name "*page*" -o -name "*view*" | grep -E "\.(dart|tsx|jsx)$" | head -200
   
2. تحقق من تناسق التصميم:
   - هل يوجد Design System / Theme مركزي؟
   - هل الألوان معرّفة في مكان واحد؟
   - هل الخطوط متسقة؟
   - هل المسافات (spacing/padding) موحدة؟
   - هل يوجد مكونات مشتركة (Shared Widgets/Components)؟

3. تحقق من دعم RTL:
   - grep -rn "textDirection\|TextDirection\|Directionality\|dir=" --include="*.dart" --include="*.tsx"
   - هل كل الشاشات تدعم RTL بشكل صحيح؟
   - هل الأيقونات معكوسة حيث يلزم؟
   - هل المحاذاة صحيحة (start/end بدل left/right)؟
   - ابحث عن left/right hardcoded: grep -rn "EdgeInsets\.\(only\|fromLTRB\)\|Alignment\.\(center\|top\|bottom\)Left\|Alignment\.\(center\|top\|bottom\)Right\|textAlign.*TextAlign\.\(left\|right\)" --include="*.dart"

4. تحقق من حالات الشاشات:
   - هل يوجد Loading State لكل شاشة؟
   - هل يوجد Empty State (لا توجد بيانات)؟
   - هل يوجد Error State مع زر إعادة المحاولة؟
   - هل يوجد skeleton loaders؟

5. تحقق من التجاوب:
   - هل التطبيق يعمل على أحجام شاشات مختلفة؟
   - هل يوجد breakpoints للتابلت؟
   - هل الجداول scrollable على الشاشات الصغيرة؟

6. تحقق من سهولة الاستخدام:
   - هل أزرار اللمس بحجم كافي (48x48 minimum)؟
   - هل يوجد تأكيد قبل العمليات الحرجة (حذف، دفع)؟
   - هل يوجد Snackbar/Toast للتنبيهات؟
   - هل التنقل سهل وواضح؟
```

### الوكيل 3.2 — مراجعة الأداء (Performance Audit)
```
قم بالتالي:

1. فحص إعادة البناء غير الضرورية (Flutter):
   - ابحث عن setState في مكونات كبيرة
   - ابحث عن عدم استخدام const constructors: grep -rn "new " --include="*.dart" | head -50
   - ابحث عن builders بدون keys
   - grep -rn "setState" --include="*.dart" | wc -l

2. فحص الصور والأصول:
   - find . -name "*.png" -o -name "*.jpg" -o -name "*.jpeg" -o -name "*.svg" -o -name "*.gif" | xargs ls -lh 2>/dev/null | sort -k5 -h -r | head -20
   - هل الصور محسّنة؟
   - هل يوجد lazy loading للصور؟
   - هل يوجد caching للصور؟

3. فحص طلبات الشبكة:
   - هل يوجد caching لاستجابات API؟
   - هل يوجد pagination للقوائم الكبيرة؟
   - هل يوجد debounce على البحث؟
   - هل يوجد retry logic؟
   - ابحث عن طلبات متكررة غير ضرورية

4. فحص الذاكرة:
   - هل Controllers/Streams يتم dispose بشكل صحيح؟
   - grep -rn "dispose\|close\(\)" --include="*.dart" | head -50
   - هل يوجد listeners لم يتم إزالتها؟
   - هل يوجد حلقات لا نهائية محتملة؟

5. فحص حجم التطبيق:
   - عدد المكتبات المستخدمة
   - هل يوجد مكتبات غير مستخدمة؟
   - هل tree shaking مفعّل؟

6. فحص الأداء العام:
   - هل يوجد heavy computations على الـ main thread؟
   - هل يوجد استخدام لـ Isolates/Web Workers للعمليات الثقيلة؟
   - هل الـ animations بـ 60fps؟
```

### الوكيل 3.3 — الوضع بدون إنترنت والتخزين المؤقت
```
قم بالتالي:

1. تحقق من الوضع Offline:
   - هل التطبيق يعمل بدون إنترنت؟
   - هل البيانات مخزنة محلياً؟
   - هل يوجد مزامنة عند عودة الاتصال؟
   - ماذا يحدث لو انقطع الإنترنت أثناء عملية دفع؟

2. تحقق من التخزين المؤقت:
   - هل يوجد local database (Hive/SQLite/SharedPreferences)؟
   - هل البيانات المتكررة مخزنة مؤقتاً؟
   - هل يوجد سياسة invalidation للكاش؟
```
