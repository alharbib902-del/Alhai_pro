# Terminal 2 — Security & Database Audit

أنت خبير أمن معلومات ومتخصص في قواعد البيانات.

## الصلاحيات
- لك صلاحية كاملة لقراءة جميع الملفات في المشروع
- لك صلاحية تشغيل أي أمر فحص أمني
- ممنوع حذف أو تعديل أي ملف
- يمكنك تثبيت أدوات فحص أمني فقط

## المهام المطلوبة

### الوكيل 2.1 — فحص الحماية (Security Audit)
```
قم بالتالي:

1. ابحث عن أسرار مكشوفة:
   grep -rn "api_key\|apiKey\|API_KEY\|secret\|SECRET\|password\|PASSWORD\|token\|TOKEN\|supabase_url\|SUPABASE\|firebase\|FIREBASE" --include="*.dart" --include="*.ts" --include="*.tsx" --include="*.js" --include="*.env" --include="*.json" --include="*.yaml" --include="*.yml" .
   
2. تحقق من ملفات .env و .gitignore:
   - هل .env مضاف في .gitignore؟
   - هل يوجد .env.example بدون قيم حقيقية؟
   - هل يوجد مفاتيح في الكود المصدري مباشرة؟

3. فحص المصادقة والتخويل:
   - كيف يتم تخزين JWT tokens؟
   - هل يوجد token refresh logic؟
   - هل يوجد session timeout؟
   - هل كل الـ endpoints محمية؟

4. فحص تعقيم المدخلات:
   - هل المدخلات مفلترة ضد SQL Injection؟
   - هل يوجد حماية XSS؟
   - هل الملفات المرفوعة يتم فحصها؟
   - هل يوجد rate limiting؟

5. فحص ZATCA Integration Security:
   - كيف يتم تخزين شهادات ZATCA؟
   - هل الاتصال بـ ZATCA عبر HTTPS فقط؟
   - هل يوجد certificate pinning؟

6. فحص التبعيات:
   - Flutter: dart pub outdated
   - React: npm audit 2>/dev/null
   - ابحث عن مكتبات بها ثغرات معروفة
```

### الوكيل 2.2 — فحص قاعدة البيانات (Database Audit)
```
قم بالتالي:

1. ابحث عن ملفات الهجرة والمخططات:
   find . -path "*/migration*" -o -path "*/schema*" -o -name "*.sql" -o -path "*/supabase*" | head -100

2. تحقق من Supabase RLS:
   - هل Row Level Security مفعّل على كل الجداول؟
   - هل السياسات مكتوبة بشكل صحيح؟
   - هل يوجد جداول بدون RLS (مكشوفة)؟

3. تحقق من تصميم قاعدة البيانات:
   - هل العلاقات معرّفة بـ Foreign Keys؟
   - هل يوجد indexes على الأعمدة المستخدمة في البحث والفلترة؟
   - هل يوجد N+1 query problem؟
   - هل يوجد استعلامات بطيئة محتملة؟

4. تحقق من Multi-tenancy:
   - كيف يتم عزل بيانات المستأجرين؟
   - هل يمكن لمستأجر الوصول لبيانات مستأجر آخر؟
   - هل الـ tenant_id موجود في كل استعلام؟

5. تحقق من سلامة البيانات:
   - هل يوجد Soft Delete حيث يلزم؟
   - هل المعاملات المالية atomic (transactions)؟
   - هل يوجد audit trail / logging؟
   - هل يوجد استراتيجية backup؟

6. ابحث عن مشاكل الأداء:
   - استعلامات بدون LIMIT
   - SELECT * بدلاً من تحديد الأعمدة
   - عمليات في loops بدون batching
```

### الوكيل 2.3 — فحص التشفير وحماية البيانات
```
قم بالتالي:

1. تحقق من تشفير البيانات:
   - هل البيانات الحساسة مشفرة at rest؟
   - هل الاتصالات مشفرة in transit (HTTPS)؟
   - هل كلمات المرور مُهشّرة (bcrypt/argon2)؟

2. تحقق من حماية البيانات الشخصية:
   - هل يوجد سياسة خصوصية؟
   - هل البيانات الشخصية محمية وفق الأنظمة السعودية؟
   - هل يمكن حذف بيانات العميل عند الطلب؟

3. تحقق من أمان التخزين المحلي:
   - هل البيانات المخزنة محلياً مشفرة (SecureStorage)؟
   - هل التطبيق يمسح البيانات عند تسجيل الخروج؟
   - هل يوجد حماية ضد root/jailbreak؟
```
