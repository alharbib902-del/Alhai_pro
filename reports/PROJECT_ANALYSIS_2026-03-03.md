## 6. البنية التقنية (Architecture)

### 6.1 إدارة الحالة (State Management)
- **Riverpod** (`flutter_riverpod`) - إدارة حالة تفاعلية
- **StateNotifier** للحالات المعقدة (مثل السمة)
- **FutureProvider / StreamProvider** للبيانات غير المتزامنة

### 6.2 قاعدة البيانات
```
┌─────────────────────────────────┐
│        التطبيق (Flutter)         │
├─────────────────────────────────┤
│     Drift ORM (SQLite/WASM)     │  ← قاعدة بيانات محلية
├─────────────────────────────────┤
│       Sync Engine (Queue)       │  ← محرك المزامنة
├─────────────────────────────────┤
│    Supabase (PostgreSQL)        │  ← قاعدة بيانات سحابية
└─────────────────────────────────┘
```

### 6.3 المصادقة
```
┌──────────────────┐
│   إدخال الهاتف   │ → check_cashier_by_phone (RPC)
├──────────────────┤
│   إدخال OTP     │ → signInWithEmailPassword
├──────────────────┤
│   اختيار المتجر  │ → store_members query
├──────────────────┤
│   الشاشة الرئيسية │
└──────────────────┘
```
- الأدوار المدعومة: `cashier`, `employee`, `store_owner`, `super_admin`
- OTP التطوير: `123456`

### 6.4 التوجيه (Routing)
- **GoRouter** مع ShellRoute لهيكل التنقل المستمر
- Nested Routes للتنقل المعقد في Admin
- Bottom Navigation في Admin Lite
- حماية المسارات (Route Guards) حسب الصلاحيات

### 6.5 التوطين (i18n)
- 7 لغات: `ar`, `en`, `ur`, `hi`, `bn`, `fil`, `id`
- ملفات ARB مع أكثر من **1,200 مفتاح ترجمة**
- دعم RTL/LTR تلقائي
- خادم AI يدعم نفس اللغات السبع

### 6.6 خدمة WhatsApp
```
┌──────────────────┐
│ WhatsApp Service  │ → Queue-based messaging
├──────────────────┤
│ WaSender Client   │ → REST API calls
├──────────────────┤
│ wasenderapi.com   │ → يرسل عبر WhatsApp Business
└──────────────────┘
```
- التهيئة عبر `--dart-define` أثناء البناء
- طابور رسائل مع إعادة المحاولة
- تنسيق أرقام سعودية تلقائي
- قوالب رسائل مخصصة
