## Cashier

**الحالة الحالية**

تطبيق نقطة البيع هو الأكثر نضجاً في المنصة، مع 505 اختبار ناجح وتكامل Sentry واختبارات E2E وintegration. عملياً هو الأقرب للجاهزية التشغيلية، لكن الجاهزية الإنتاجية النهائية ما زالت متوقفة على البناء والتوقيع والهوية البصرية وتهيئة أسرار الإنتاج.

**عدد الشاشات**

- 47 ملف شاشة محلي داخل `apps/cashier/lib/screens`
- 93 route في `apps/cashier/lib/router/cashier_router.dart`
- 79 شاشة وظيفية موثقة في README العام

**أهم المشاكل المفتوحة**

- فشل `flutter build apk --debug` محلياً ما لم يكن إصلاح Kotlin imports قد طُبق.
- لا يوجد release keystore فعلي.
- مجلد `ios/` غير موجود.
- أيقونة Flutter الافتراضية ما زالت مستخدمة.
- النسخة محبوسة عند `1.0.0-beta.1+1`.
- `SENTRY_DSN_CASHIER` غير مضمون في GitHub Secrets.

## Admin

**الحالة الحالية**

لوحة الإدارة الكاملة مستقرة إلى حد جيد، مع 361 اختبار ناجح ودمج حديث لـ Sentry، لكنها ما زالت في مرحلة hardening وليست “جاهزة للإطلاق” بدون معالجة build والنواقص التشغيلية وبعض الديون التقنية legacy.

**عدد الشاشات**

- 62 ملف شاشة محلي داخل `apps/admin/lib/screens`
- 122 route في `apps/admin/lib/router/admin_router.dart`
- 106 شاشة موثقة على مستوى المنتج

**أهم المشاكل المفتوحة**

- Android build محلي يحتاج تحقق بسبب مشكلة Kotlin imports.
- لا يوجد مشروع iOS.
- 6 مواضع `catch (_) {}` legacy ما زالت موجودة.
- `media_library_screen.dart` يحتوي placeholder strings.
- أيقونة Flutter الافتراضية ما زالت موجودة.
- النسخة ثابتة عند `1.0.0-beta.1+1`.

## ZATCA Package

**الحالة الحالية**

هذه الحزمة هي الأعلى من ناحية الثقة التقنية داخل المنصة: 812 اختبار ناجح + 1 skipped، وإصلاح bug مهم في ECDSA parser تم توثيقه باختبارات regression. تقنياً هي قوية، لكن التسليم النهائي الإنتاجي يعتمد على onboarding الفعلي والشهادات وليس على الكود فقط.

**عدد الشاشات**

- لا توجد شاشات UI
- 37 source modules أساسية في `lib/src`

**أهم المشاكل المفتوحة**

- CSID onboarding يحتاج تنفيذ فعلي لكل متجر.
- لا توجد production certificates محمّلة مسبقاً.
- بعض special invoice types تحتاج اختبارات إضافية.
- offline queue تحتاج monitoring تشغيلي عند الإنتاج.

**الوظائف الأساسية داخل الحزمة**

| الوحدة | الوظيفة |
|---|---|
| API clients | clearance, compliance, reporting |
| CSR generator | توليد CSR |
| Certificate storage | تخزين الشهادات |
| Certificate renewal | متابعة تجديد الشهادات |
| Chain store / invoice chain | إدارة PIH hash chain |
| QR service | توليد QR وفق TLV |
| VAT calculator | حساب VAT |
| ECDSA signer | التوقيع الرقمي |
| XAdES signer | تغليف التوقيع في XAdES |
| Invoice hasher | حساب hash الفاتورة |
| UBL builders | توليد XML UBL 2.1 |
| Compliance checker | التحقق من الامتثال |
| Offline queue | حفظ الفواتير محلياً عند انقطاع الشبكة |
| Providers/DI | ربط الحزمة بالتطبيقات المستهلكة |

**ما يحتاجه قبل التسليم النهائي**

- onboarding فعلي عبر Fatoora portal لكل متجر.
- استخراج وتخزين production certificates.
- اختبار أنواع فواتير خاصة إضافية حسب الحالات التجارية.
